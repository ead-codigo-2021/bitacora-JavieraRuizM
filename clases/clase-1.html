Clase 1: Introducción al diseño front-end

En esta primera clase se presentan los objetivos del curso relacionados a conocer las herramientas de diseño de códigos de interfaz para distintos medios y soportes digitales, concentrándose en el Front-end de un proyecto digital.

Se presentan principios de tecnología de web abierta, estándares web, control de versiones, planteando además una visión crítica y reflexiva relacionada al Responsive Web Design y Resilient Web Design (como puedo producir algo que en el futuro se vea de la misma manera, utilizando como ejemplo contrario “Flash”, que funciona mediante pluggin y no con estándar web). Se trata la relación entre contenidos, datos e interfaz considerando la arquitectura de contenidos, modelamiento de datos, patrones de diseño y diseño de componentes.

¿Cómo funciona un sitio web?

Se accede a través de una URL, desde un navegador (Chrome, Safari, Firefox, etc), el cual posee un motor de renderizado que permite que el sitio se visibilice de una forma determinada.

Definiciones importantes:

  1. Front-End: interfaz, lo que ve el usuario al ingresar a un sitio
  2. Back-End: Servidor, información y base de datos
  3. HTML ( Hypertext Markup Language): se relaciona al contenido y estructura del sitio
  4. CSS (Cascading Style Sheets): otorgan estilo al contenido
  5. JS (JavaScript): nos permite interactuar con el sitio mediante eventos, funciones y acciones

Entregables y proceso de proyecto

  1. Investigación: Benchmark, personas, escenarios. Análisis del problema, competencia, identificar y reconocer a los usuarios (lenguaje comunicacional, etc), definiciones estratégicas (levantamiento de información).
  2. Contenidos: Mapa de arquitectura de contenidos. Jerarquización, ordenar lo que se verá en la pantalla, identificar el sentido.
  3. Diseño de Interacción Wirefranmes. Definir jerarquías visuales de información
  4. Front-End HTML, CSS, JS. Estándares web, semántica, accesibilidad, performance (sitio optimizado para descarga rápida)
  5. Back-End Sitio web, APIs. Seguridad, estabilidad, escalabilidad, administración. Lenguajes de programación Python, PHP.

Clase 2: HTML

En esta clase se presentó HTML, es decir Hypertext Markup Language. Es un protocolo estándar de documentos para que estos sean visibles en un navegador web, permitiendo indicar la estructura de nuestro documento mediante etiquetas. Este lenguaje nos ofrece una gran adaptabilidad y una estructuración lógica.

Indentación

Es una buena práctica referida a la aplicación de sucesivos niveles de margen de acuerdo a la anidación de etiquetas. Es bastante útil para identificar problemas de anidación o desbalance de etiquetas de cierre con mayor facilidad.

Atributos

Los atributos de HTML son palabras especiales utilizadas dentro de la etiqueta de apertura, para controlar el comportamiento del elemento.

Recomendaciones generales de escritura

Cuidar indentación, preferir minúsculas para etiquetas y atributos, comillas dobles para atributos, cerrar etiquetas, evitar líneas demasiado largas, evitar espacios en blanco innecesarios.

Estructura base de un documento HTML

Doctype: Estándar html que voy a usar

Head: Metadatos, vinculación CSS y JS. Un ejemplo de metadato es Viewport, que se refiere a la ventana disponible que tiene el usuario. Permite trabajar con diseño responsivo.

Body: Contenido visible para el usuario

Semántica HTML

Se refiere al sentido que tienen las etiquetas relacionadas a su contenido. Existe una serie de etiquetas con mayor significado, para cada parte, sección, o elemento de nuestra página, y que, aunque en la práctica no generen un resultado distinto al de usar una etiqueta

como contenedor para todo, pueden darle mayor significado a nuestro código.

Las ventajas de una buena semantica pueden implicar una mejor accesibilidad al contenido para los usuarios, puede ser considerado por motores de búsqueda para influir en un ranking de página (SEO), permite recorrer y rasterear bloques de código, sugiere datos para completar el código.


Clase 3: CSS

CSS o Cascading Style Sheets, es un lenguaje con el que se define la parte de la presentación, es decir, cómo deben mostrarse los elementos de la página, su posición, forma, espaciados, colores y en resumen, toda la parte estética. CSS no tiene ningún sentido sin HTML, igual que a día de hoy HTML sin CSS también resultaría imposible, ya que la web ha evolucionado de tal forma que su aspecto es fundamental y el CSS es la única vía para conseguir personalizarlo.

Para desarrollar con CSS tenemos que trabajar con una serie de elementos, mediante los cuales se declaran los estilos, básicamente éstos son los más importantes:

  1. Selectores, mediante los cuales podemos especificar qué elementos de la página nos estamos refiriendo
  2. Atributos, para definir qué cosas queremos estilizar sobre los selectores indicados
  3. Valores, que indican qué estilo se debe aplicar a cada atributo sobre cada selector. Los valores se expresan con unidades CSS, que sirven para cuantificar los valores (píxeles, puntos...)

      4. Recomendaciones de escritura

      • Utilizar minúsculas
      • Indentar
      • Solo un selector por línea
      • Solo una declaración por línea
      • Dejar un espacio entre el último selector y el paréntesis de corchete
      • Siempre terminar declaración con ;

Clase 5: Tipografía; Modelo de caja

Variable CSS

Las variables son propiedades definidas que contienen valores específicos que se pueden volver a utilizar en un documento. Se establecen mediante la notación de propiedades personalizadas (por ejemplo, --main-color: black;) y se acceden mediante la función var() (por ejemplo, color: var (--main-color);).

Tipografía para la web

La declaración @font-face nos permite utilizar fuentes personalizadas (webfonts), que podemos importar de otro sitio como googlefonts. Es un método para llamar a la tipografía desde nuestro pc, se declara al principio de CSS.

Formatos: OTF (para ocupar en el escritorio), Variable (archivo variable, distintos pesos en un archivo), Web (ttf, woff, woff2).

Modelo de caja

Se definen cajas rectangulares, incluyendo sus rellenos y márgenes, que son generadas para los elementos y que se disponen de acuerdo al modelo de formato visual. Se refiere a que todos los elementos están dentro de una caja, ocupando un espacio rectangular en el flujo de la página.

Opciones de display

Display-block: usa todo el ancho disponible, produce quiebre de línea, contenidos lo hacen crecer hacia abajo

Display-inline: usa solo espacio justo según su contenido, mantiene flujo del texto

Display inline-block: su tamaño se ajusta a su contenido, pero puede crecer, mantiene flujo del texto, crece horizontal y luego vertical, puede tener relleno y margen
